## Error Handling Strategy

### Error Categories

| Category | Status Code | Handling |
|----------|-------------|----------|
| Validation | 400 | Return field-specific errors |
| Authentication | 401 | Redirect to login |
| Authorization | 403 | Show permission denied |
| Not Found | 404 | Show 404 page |
| Rate Limit | 429 | Show retry-after |
| Server Error | 500 | Log to Sentry, show friendly error |
| AI Error | 502 | Retry with backoff, show status |

| Condition | Severity | Action |
|-----------|----------|--------|

| Error rate > 5% | Critical | PagerDuty alert |
| p95 latency > 2s | Warning | Slack notification |
| AI service down | Critical | PagerDuty + fallback |
| Queue depth > 1000 | Warning | Auto-scale workers |

### Deployment Strategy

- **Preview Deploys**: Every PR gets a preview URL
- **Staging**: Main branch auto-deploys to staging
- **Production**: Manual promotion from staging
- **Rollback**: One-click rollback to previous deployment
- **Feature Flags**: PostHog for gradual rollout
